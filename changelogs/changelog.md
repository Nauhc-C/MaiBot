# 更新日志

# [1.0.0] - 待发布

## 用户感知功能侧

### Maisaka / 回复与行为

- Maisaka 主回复链路全面升级，回复流程统一经过节奏判断、规划、工具调用、回复生成和多轮内部思考，群聊与私聊链路更一致。
- 支持多模态上下文与图片处理，模型会按视觉能力决定是否携带图片本体，并限制长对话中的图片数量，降低请求压力。
- 新增中期记忆与聊天摘要能力，可用压缩摘要扩展长对话上下文，并在重启后恢复部分上下文。
- 回复频率支持设置为 `0`，可静默接收消息并维护历史；动态发言频率规则支持按聊天流和时间段精细配置。
- 行为表现学习升级为“场景-动作-结果”路径，支持行为反馈评分、场景图谱、相似行为整合和用进废退维护，让麦麦在不同场景下选择更贴合的行为。
- 表达方式选择与审核继续优化，支持按聊天流筛选审核记录、人工恢复误拒表达，并提升表达、黑话和语境匹配稳定性。

### A_Memorix / 长期记忆

- A_Memorix 长期记忆系统主线化，支持段落、实体、关系、来源、向量和图谱统一存储，替代旧长期记忆链路。
- 长期记忆检索融合向量、图关系、BM25、PageRank、后验图检索和阈值过滤，提升召回质量和可解释性。
- 支持历史聊天总结导入长期记忆，Web Import 并发导入、来源记录、导入清单校验和按来源失效流程更完整。
- 人物画像能力增强，支持人物事实写回证据、画像自动注入、证据查看与纠错闭环。
- 长期记忆控制台、知识图谱、证据视图和维护任务体验优化，导入、维护、删除、纠错等流程更可观察。

### WebUI / Dashboard

- Dashboard 进入新的界面体系，覆盖聊天、配置、插件、记忆、知识库、统计、监控、日志、推理过程和系统设置等核心功能。
- WebUI 主题系统、移动端适配、侧边栏、按钮、表单、弹窗、日志查看器和推理过程页面持续优化，整体观感和可用性提升。
- 配置页面支持更复杂的动态表单、数字草稿输入、列表、JSON、extra params、模型任务配置和插件原始 TOML 编辑。
- 插件市场和插件管理增强，支持安装、启停、配置、更新、运行状态、README、分类、图标、评价和随机推荐展示。
- 本地缓存管理增强，支持查看和清理数据库、图片缓存、表情包缓存、日志目录和 data 目录，并支持数据库表大小统计、按表清理与 VACUUM。
- 推理过程页面支持查看阶段、工具调用、prompt 预览、请求模型、推理耗时、动作摘要和连续导航，便于排查模型行为。
- 聊天页支持图片消息、复杂消息查看、用户模拟、平台模拟和更稳定的 WebSocket 通信。
- 新增人设生成器，可从文段、角色卡或说明生成可注入的人格、表达风格和聊天提示词配置块。

### 插件 / MCP / 工具

- 插件系统重构为独立 `plugin_runtime`，支持插件独立启动、停止、重载、运行状态追踪、组件注册、事件分发、Hook、API 和 RPC。
- 插件组件支持按群聊类型、`session_id`、黑名单和白名单控制启用范围，并可声明超时时间、插件类型、图标和核心工具能力。
- 插件可访问宿主消息、聊天流、配置、运行时数据、embedding 能力和 LLM provider 适配能力，插件开发面更完整。
- MCP 能力主线化，支持加载 MCP 工具、Prompt、Resource，并可通过 Host LLM Bridge 反向调用主程序模型。
- 工具系统统一为延迟发现和索引展开模式，减少一次性塞入过多工具说明，提高工具调用稳定性。

### 表情包 / 图片 / 多模态

- 表情包管理升级，支持认识、不认识、据为己用、丢弃等状态视角，上传、注册、删除、替换和批量处理流程更可靠。
- 图片缓存新增自动清理、分页浏览、预览、按日期筛选、单个删除和批量删除能力。
- 入站图片支持过大图片压缩或丢弃，图片、工具返回媒体和转发消息在上下文中保留更稳定。
- `send_image` 等图片发送能力优化，模型可通过上下文消息或工具媒体索引更稳定地发送图片。

### 性能、稳定性与安全

- 启动流程优化，延后非关键服务初始化并减少阻塞步骤，提升启动速度。
- 统计系统改为聚合服务，降低大数据量统计的内存占用，并新增缓存率等统计信息。
- WebUI 安全加固，增强认证、路径校验、URL 校验、反爬中间件和静态资源访问防护。
- 本地存储写入、图片缓存写入、插件更新、配置升级、数据库迁移和 WebUI 重启流程更稳。
- 日志系统增加上限和清理能力，减少长期运行时的磁盘压力。

## 开发侧 / SDK / API 改动

### 架构与数据

- 核心运行结构重构，新增 `src/services` 服务层，统一 LLM、生成器、发送、消息、数据库、记忆、HTML 渲染和 Embedding 等能力。
- 新增统一 `platform_io` 消息平台抽象，入站消息、出站消息、去重、回执更新、插件驱动和旧适配器兼容走同一套链路。
- 新增数据库迁移管理器和聊天会话 `account_id`、`scope` 等归属字段，资源归属语义更清晰。
- 配置系统加强旧配置升级、字段校验、版本管理和确认流程，避免静默改写用户配置。

### 插件 SDK / Runtime

- 插件运行时新增能力注册、组件注册、事件分发、Hook 分发、API 注册、Supervisor、Runner、RPC Server/Client 等基础设施。
- 插件 Action、Command、Tool、API 和插件平台驱动支持组件元数据 `timeout_ms`，也可由能力 API 设置超时时间覆盖。
- 插件 manifest 校验更严格，支持插件类型、图标、核心工具能力、模型 provider、配置入口、README 和市场信息展示。
- 插件更新支持无独立 `.git` 的本地目录，更新前备份旧目录，失败时恢复旧版本。

### WebUI API / 调试

- WebUI 后端拆分为 app、依赖、中间件、routers、schemas、services、utils 等结构，并统一 API 响应模型与错误格式。
- 新增或增强聊天、配置、插件、表情包、表达方式、黑话、记忆、知识库、统计、系统、本地缓存、推理过程、人设生成器等 API。
- Prompt 预览支持 HTML/JSON、图片缓存引用、模型名与耗时元数据，避免默认内联大量 base64。
- WebSocket 连接管理统一，聊天、日志、状态和系统事件共用推送机制。

### Prompt / 模型调用 / 测试

- Prompt 模板继续按中文、英文、日文同步维护，新增或拆分表情包选择、中期记忆摘要、timing gate、replyer 等模板。
- LLM 服务层支持任务模型选择、单次指定模型、模型缓存、视觉能力判断、工具调用兼容和 Host LLM Bridge。
- 补充动态发言频率、本地缓存写入、插件更新、A_Memorix 导入、人物画像纠错、上下文工具、Planner 快照、prompt fallback、图片处理等回归测试。
- 开发容器、Docker、依赖锁定、Crowdin、多语言资源、CodeRabbit、PR 模板和调试脚本等工程设施同步更新。
